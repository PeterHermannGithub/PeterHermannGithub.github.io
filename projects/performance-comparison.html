Jaccard similarity measures the similarity between two finite sets by dividing the size of their intersection by the size of their union. In our anime recommendation system, this algorithm computes similarity between anime based on shared tags and genres.

The Challenge

While building an anime recommendation system, I encountered a critical performance bottleneck: computing Jaccard similarity for a 3,000 × 3,000 matrix was taking 9 seconds per operation. For a responsive user experience, this needed to be reduced to under 100 milliseconds.

This project explores three different approaches to the same computational problem: a Python baseline for rapid prototyping, a C++ implementation with OpenMP parallelization, and a Rust solution leveraging Rayon for safe concurrency. The goal was to achieve a 50-100x performance improvement while maintaining identical computational results.

Optimization Strategies

Parallel Processing

Leveraged OpenMP in C++ and Rayon in Rust for efficient multi-core utilization

Memory Optimization

Symmetric matrix computation (upper triangle only) and cache-friendly data structures

Algorithmic Improvements

Vectorized operations, binary set representations, and early termination conditions

Implementation Highlights

Rust Implementation

use rayon::prelude::*;

pub fn compute_similarity_matrix_parallel(
    anime_tags: &[Vec<u32>], 
    tag_count: u32
) -> Vec<Vec<f32>> {
    let n = anime_tags.len();
    let mut matrix = vec![vec![0.0; n]; n];
    
    matrix.par_iter_mut().enumerate().for_each(|(i, row)| {
        for j in (i + 1)..n {
            let similarity = jaccard_similarity_binary(
                &anime_tags[i], 
                &anime_tags[j], 
                tag_count
            );
            row[j] = similarity;
        }
    });
    
    matrix
}

C++ Implementation

#pragma omp parallel for schedule(dynamic)
for (int i = 0; i < n; ++i) {
    for (int j = i + 1; j < n; ++j) {
        float similarity = jaccardSimilarity(
            animeData[i], 
            animeData[j]
        );
        matrix[i][j] = similarity;
        matrix[j][i] = similarity; // Symmetric
    }
}

Key Learnings

  • Memory layout optimization can be as important as algorithmic improvements
  • Rust's ownership model enables safe parallelism without runtime overhead
  • Profile-guided optimization revealed unexpected bottlenecks
  • Language choice should be driven by performance requirements and team expertise
